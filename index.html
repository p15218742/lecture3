# CSS good practice - **1** **definites** - The universal selector **`*`** is slow - Avoid **`!important`**. It means the specificity of your CSS is out of control - No trailing zeros in values e.g. **`2em`** not **`2.0em`** - Use hyphens, not underscores, for multi-word ID or class names - After general styles, order styles as they appear in the source code

# Responsive design: **0** - mobile first - responsive design and breakpoints **Books** [Luke Wroblewski, 'Mobile First’](http://abookapart.com/products/mobile-first) [Ethan Marcotte, 'Responsive Web Design’](http://abookapart.com/products/responsive-web-design)
# Responsive design: **1** ## Natural breakpoints The best approach to responsive design uses [natural breakpoints](http://stackoverflow.com/questions/6370690/media-queries-how-to-target-desktop-tablet-and-mobile/20350990#20350990) over fixed device screen sizes. Design **mobile-first**, then increase the screen width. Add **breakpoints** at the viewport sizes where the design becomes "broken".
# Responsive design: **2** ## Percentages Percentages work well for setting the element widths, as long as you use **`display: border-box`** to ensure padding and borders remain *inside* the element’s width.
