# JAVASCRIPT EVOLUTION: **1** was **created in 10 days** in May **1995** by [Brendan Eich](https://brendaneich.com/), then working at Netscape and now Mozilla. JavaScript was not always known as JavaScript: the original name was **Mocha**, a name chosen by [Marc Andreessen](http://www.biography.com/people/marc-andreessen-9542208), founder of [Netscape](https://www.engadget.com/2014/05/10/history-of-netscape/). In 1995 the name became **LiveScript** then (after a trademark license from [Sun](https://www.oracle.com/sun/)) to **JavaScript**—a **marketing move** as Java was then very popular and was used in browsers. As a consequence, beginners still **confuse** the two languages. But **JavaScript IS NOT Java**! # JAVASCRIPT: **3** ## How does JavaScript work? 1. builds an **execution stack** to handle your code 2. executes your code **synchronously** (one thing at a time) 3. the browser adds user events (e.g. click) to an **event queue** 4. if the **execution stack** is empty… 5. …JavaScript **checks** the event queue for **actionable events** --- The browser does things *asynchronously* (**rendering the page**, making **HTTP calls**, storing **user events**…) while JavaScript handles things *synchronously*
# JAVASCRIPT: **4** ## THE **EVENT LOOP** For interaction in the browser, JavaScript runs in an **event loop**, so you can think in terms of **event handlers** that **listen** for events triggered by the user: ``` javascript let myElement = document.getElementById("my-id") myElement.addEventListener("event", do_stuff); // myElement refers to the HTML tag with an id "my-id", // the "event" could be "submit", "click", etc. // do_stuff can be a function called after the event ``` --- **CSS** can be triggered by focus/hover/checked etc., **JavaScript** can listen for **hundreds** of events: [Web Event reference (MDN)](https://developer.mozilla.org/en-US/docs/Web/Events)

    # JAVASCRIPT SYNTAX: **7** ### Array array literal - create and populate: ```javascript var myArray = [1, 2, "free", multiplyNums(8, 8)]; myArray[3]; // 64 ``` **No comma** after the **last element**! ```javascript var myArray = [1, 2, "free", multiplyNums]; myArray[3](2, 2); // 4 ``` **No brackets** in a function as an array element!
    # JAVASCRIPT SYNTAX: **8** ### Associative array (object) Contains **key value** pairs. This is declared as an 'object literal': ```javascript var myObject = {key1: 64, key2: "text"}; ``` Access/change: **square brackets** or **dot** (property) syntax: ```javascript alert(myObject['key2']); alert(myObject.key2); myObject.key2 = 'More text'; ``` add a new key/value pair: ```javascript myObject.key3 = Math.PI; ```
    # JAVASCRIPT SYNTAX: **9** Object keys look like names (like variables), but the *key is a string*, which allows expressions like this: ```javascript console.log(myObject["key"+2]); console.log(myObject["key2"]); // same as above ``` By the relaxed rules of JavaScript data typing `"key"+2` means `"key"+"2"` which evaluates to the string `"key2"`. So you can also specify keys as a string (like JSON): ```javascript var myObject = {"key1": 64, "key2": "text"}; ```
