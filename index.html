# CSS ANIMATION: **1** The [CSS `animation` property](http://www.w3schools.com/cssref/css3_pr_animation.asp) binds a custom `animation` to an element identified by it’s `selector` e.g. ```css .see-this { animation: change-stuff 5s infinite; } ``` (animation name, duration, how many times to run) `infinite` runs forever, seconds/milliseconds limit time The changes are user-defined in a separate `@keyframes` code block
# CSS ANIMATION: **2** ## KEYFRAMES ```css @keyframes change-stuff { 25% { styles for 1/4 way through; } 50% { styles for 1/2 way through; } 75% { styles for 3/4 way through; } } ``` See [W3Schools CSS3 Animations](http://www.w3schools.com/css/css3_animations.asp)
# CSS ANIMATION: **3** ## COMPONENTS - **transitions** (things to change, has *timing*) - **transformations** (shape-shifting, *no* timing) - **keyframes** (composing *custom* timelines) use separately or together [Early use of `transition` Dan Cederholm (2010)](http://alistapart.com/article/understanding-css3-transitions)
# CSS ANIMATION: **4** ## TRANSITION changes CSS properties over a given time e.g. ```css transition: width 4s ease-in-out 1s; /* cubic-bezier(0.6, -0.28, 0.735, 0.045) */ ``` **property**: the CSS to transition (comma-separated, all) **duration**: length of transition (secs/millisecs `s` `ms`, `infinite`) **timing-function**: (optional) ‘speed curve’ of the transition **delay**: (optional) time before transition starts OMG TMI: [Easing functions](http://easings.net/) LESS i: [CSS3 transition Property (W3Schools)](http://www.w3schools.com/cssref/css3_pr_transition.asp)
# CSS ANIMATION: **5** ## TRANSFORM include in the element you want to transform, then change styles on hover/whatever ```css transform: rotate(70deg); ``` **translate**: `translate(540px,-200px);` x, y **rotate** `rotate(360deg);` **scale** `scale(2, 0.5);` x, y **skew** `skew(10deg);` x axis **matrix** `matrix(1, -0.3, 0, 1, 0, 0)` scaleX,skewY,skewX,scaleY,translateX,translateY
# CSS ANIMATION: **6** ## what is (not) animatable? **not** `display: none|block` [Animatable CSS properties (W3Schools)](http://www.w3schools.com/CSSref/css_animatable.asp) triggered by focus/hover/etc.… that can be styled e.g.: `:focus :blur :hover :active input[type=checkbox]:checked …`
# CSS ANIMATION: **7** ## ONLY IF NECESSARY “CSS animations, transforms and transitions are not automatically GPU accelerated, and instead execute from the browser’s slower software rendering engine. So what exactly forces the browser to swap to GPU mode? Many browsers provide GPU acceleration by means of certain CSS rules.” — [Guil Hernandez (2012)](http://blog.teamtreehouse.com/increase-your-sites-performance-with-hardware-accelerated-css) - [CSS `will-change` is a last resort](https://developer.mozilla.org/en/docs/Web/CSS/will-change) - [detailed hardware acceleration answer on StackOverflow](http://stackoverflow.com/a/26371807/123033)
