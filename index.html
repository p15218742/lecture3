# JAVASCRIPT EVOLVES: **1** was **created in 10 days** in May **1995** by [Brendan Eich](https://brendaneich.com/), then working at Netscape and now of Mozilla. JavaScript was not always known as JavaScript: the original name was **Mocha**, a name chosen by [Marc Andreessen](http://www.biography.com/people/marc-andreessen-9542208), founder of [Netscape](https://www.engadget.com/2014/05/10/history-of-netscape/). In September 1995 the name changed to **LiveScript**, then in December (after a trademark license from [Sun](https://www.oracle.com/sun/)) to JavaScript—a **marketing move** at that time as Java was very popular. As a consequence, beginners have **confused** the two ever since. [A Short History of JavaScript (W3C)](https://www.w3.org/community/webed/wiki/A_Short_History_of_JavaScript)
# JAVASCRIPT EVOLVES: **2** - **1995:** JavaScript born as Mocha/LiveScript - **1997:** ECMAScript standard established ([see ECMA](http://www.ecma-international.org/ "Standardization of Information and Communication Technology and Consumer Electronics: an international private (membership-based) non-profit standards organization for information and communication systems")) - **1999:** ES3 - Microsoft discover www. [Crockford](http://javascript.crockford.com/ "…and his JavaScript stuff")), and standard [JSON](http://www.w3schools.com/js/js_json_intro.asp "JSON: the nice introduction") ([JSON motherlode here](http://www.json.org/ "A big load of information about the rather unfortunately-numbered ECMA-404")) - **2015:** ES6/ECMAScript2015 - mostly syntactic sugar, because no firm agreement on anything more radical [Condensed from benmccormick.org](https://benmccormick.org/2015/09/14/es5-es6-es2016-es-next-whats-going-on-with-javascript-versioning/)

# JAVASCRIPT EVOLVES: **5** ## JQuery (a library) Although we're not teaching it (various reasons), you will encounter [JQuery](https://jquery.com/)—a JavaScript library created from an [original concept by John Resig in 2005](http://ejohn.org/blog/selectors-in-javascript/) and [announced in 2006 at BarCampNYC](http://ejohn.org/blog/barcampnyc-wrap-up/). It simplified how JavaScript interfaces with the DOM—one feature being the ability to select elements using CSS syntax: ```javascript $( "#button-container button" ).on( "click", function( event ) { hiddenBox.show(); }); ``` Some web frameworks include JQuery (or a *light* version) alongside their own libraries.

# JAVASCRIPT: **1** ## The browser event loop Inside the browser, everything runs in an **event loop** so you think in terms of **event handlers** that **listen** for events: ``` javascript myElement.addEventListener("id-name", do_stuff); ``` While CSS can be triggered by focus/hover/etc., JavaScript can listen for **hundreds** of events: Massive list: [Web Event reference (MDN)](https://developer.mozilla.org/en-US/docs/Web/Events)

# JAVASCRIPT: **3** **var** and **let** ```javascript function demoLet() { //tuce is *not* visible out here for( let tuce = 0; tuce < 5; tuce++ ) { //tuce is only visible in here (and in the for()) } //tuce is *not* visible out here } function demoFor() { //nish *is* visible out here for( var nish = 0; nish < 5; nish++ ) { //nish is visible to the whole function } //nish *is* visible out here } ``` […difference between using “let” and “var” to declare a variable?](http://stackoverflow.com/a/11444416/123033)
# JAVASCRIPT: **4a** ## OBJECTS: **2** A [JavaScript object](http://www.w3schools.com/js/js_objects.asp) contains a series of colon-separated name-value pairs (no comma after the last one!!) enclosed in curly braces `{}`: ```javascript var car = { manufacturer: "Tesla", model: "Model S", range: "300", colour: "red", start: function() {startMyCarStuffHere} }; ``` You can have an **array of objects**, which is similar to a common data format in JSON.

# JAVASCRIPT: **5** ## JSON: **1** [JSON](http://www.json.org/) is a compact **portable data format**, readable in many languages, added to databases, translated into/from (say) XML… It is used in many APIs where you get the data from a remote server (say weather), and that data is recieved as JSON. [JSON Placeholder has some good examples](https://jsonplaceholder.typicode.com/posts/1/comments) and can be used for development or learning. An **array** of **objects** is a common format.
